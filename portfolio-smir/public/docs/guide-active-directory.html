1. Active Directory, c'est quoi ?

Active Directory (AD) est le service d'annuaire de Microsoft. C'est une base de données centralisée qui stocke et organise l'ensemble des informations d'un réseau d'entreprise :

  • Utilisateurs et mots de passe
  • Ordinateurs et imprimantes
  • Groupes et permissions
  • Stratégies de groupe (GPO)

Pourquoi utiliser Active Directory ?

1 Authentification centralisée
2 Gestion des droits
3 Déploiement GPO
4 Organisation réseau
  • Centraliser l'authentification : un seul compte utilisateur pour tous les PC du domaine
  • Gérer les droits d'accès : définir qui a accès à quoi (dossiers, imprimantes, applications)
  • Déployer des GPO : appliquer des configurations et restrictions à distance
  • Organiser le réseau : structurer les ressources en unités d'organisation (OU)

Un seul compte AD permet à un utilisateur de se connecter sur n'importe quel poste du domaine avec le même identifiant et mot de passe.

2. Principaux éléments

Domaine

Unité de base de l'AD. Chaque domaine possède un nom unique (ex: entreprise.local) et contient un ensemble d'objets (utilisateurs, ordinateurs, etc.).

Forêt (Forest)

Ensemble de domaines partageant un catalogue global commun. La forêt est la frontière de sécurité la plus haute dans AD.

Unité d'Organisation (OU)

Conteneur logique permettant d'organiser les objets du domaine en arborescence hiérarchique. Les OU servent à structurer l'annuaire et à appliquer des GPO ciblées.

Exemple d'arborescence OU

CPMS (Domaine racine) |-- Utilisateurs | |-- RH | |-- IT | |-- Direction |-- Ordinateurs | |-- Laptops | |-- Fixes |-- Serveurs

Objets

Tout élément stocké dans l'annuaire AD est un objet :

  • Utilisateurs
  • Groupes (sécurité ou distribution)
  • Ordinateurs
  • Imprimantes
  • Partages réseau
  • Stratégies de groupe (GPO)

Contrôleur de Domaine (DC)

Serveur qui héberge l'annuaire AD et gère l'authentification des utilisateurs. Un domaine nécessite au minimum 1 DC, mais il est recommandé d'en avoir au moins 2 pour assurer la redondance et la continuité de service.

Un seul contrôleur de domaine est un point de défaillance unique (SPOF). Prévoyez toujours un second DC pour la tolérance de pannes.

3. Installation AD DS

Prérequis

  • Windows Server 2019 ou 2022 installé
  • Adresse IP statique configurée
  • Nom du serveur défini (ex: SRV-AD01)

Étape 1 — Configurer l'IP statique

Ouvrir les paramètres réseau du serveur et configurer une IP fixe :

ParamètreValeur
Adresse IP192.168.1.10
Masque de sous-réseau255.255.255.0
Passerelle par défaut192.168.1.1
Serveur DNS préféré127.0.0.1

Le DNS préféré doit pointer vers 127.0.0.1 (lui-même), car le DC sera aussi serveur DNS. Cela garantit la résolution interne du domaine.

Étape 2 — Ajouter les rôles AD DS et DNS

  1. Ouvrir Server Manager (Gestionnaire de serveur)
  2. Cliquer sur Manage > Add Roles and Features
  3. Sélectionner le type d'installation : Role-based or feature-based installation
  4. Cocher les rôles : Active Directory Domain Services et DNS Server
  5. Valider et installer

Étape 3 — Promouvoir en contrôleur de domaine

  1. Après l'installation, cliquer sur la notification "Promote this server to a domain controller"
  2. Sélectionner "Add a new forest"
  3. Entrer le nom du domaine racine : entreprise.local
  4. Définir le mot de passe DSRM (Directory Services Restore Mode)
  5. Valider les options par défaut et lancer l'installation
  6. Le serveur redémarrera automatiquement

Le mot de passe DSRM est indispensable pour la restauration d'Active Directory en mode sans échec. Conservez-le en lieu sûr.

4. Configuration DNS

Active Directory nécessite obligatoirement un serveur DNS fonctionnel. Le DNS est utilisé en interne pour :

  • Localisation du contrôleur de domaine par les clients
  • Résolution de noms pour l'application des GPO
  • Enregistrement automatique des machines dans la zone DNS

Zones DNS créées automatiquement

ZoneRôle
entreprise.localZone de recherche directe principale du domaine
_msdcs.entreprise.localZone de localisation des services Microsoft (SRV records)

Le contrôleur de domaine doit toujours pointer vers lui-même comme serveur DNS préféré. Ne jamais mettre un DNS externe (8.8.8.8) en DNS principal sur le DC.

5. Installation DHCP

Ajout du rôle DHCP

  1. Ouvrir Server Manager > Add Roles and Features
  2. Cocher le rôle DHCP Server
  3. Valider et installer
  4. Autoriser le serveur DHCP dans Active Directory

Configuration de l'étendue (Scope)

ParamètreValeur
Plage d'adresses192.168.1.100 à 192.168.1.200
Adresses réservéesServeurs (plage basse, hors du scope)
Option : Routeur192.168.1.1
Option : Serveur DNS192.168.1.10
Durée du bail8 jours (par défaut)

Réservez toujours une plage d'adresses fixes pour vos serveurs et équipements réseau (imprimantes, NAS, etc.) en dehors du scope DHCP.

6. Stratégies de groupe (GPO)

Les Group Policy Objects (GPO) permettent de déployer des configurations et restrictions à grande échelle, appliquées automatiquement aux utilisateurs et ordinateurs du domaine.

Sécurité

  • Imposer des mots de passe complexes (longueur, caractères spéciaux, expiration)
  • Bloquer l'accès au Panneau de configuration
  • Désactiver les ports USB pour empêcher les copies de données

Personnalisation

  • Déployer un fond d'écran d'entreprise sur tous les postes
  • Afficher un message d'accueil (legal notice) à l'ouverture de session

Configuration système

  • Déployer des logiciels au format MSI automatiquement
  • Exécuter des scripts à l'ouverture ou la fermeture de session (logon/logoff)
  • Mapper des lecteurs réseau automatiquement

Commandes utiles GPO

Forcer la mise à jour des GPO sur un postegpupdate /force
Ouvrir la console de gestion des GPOgpmc.msc

Testez toujours vos GPO sur une OU de test avant de les déployer en production. Utilisez gpresult /r pour vérifier les GPO appliquées à un utilisateur ou un ordinateur.

7. Ajout d'un client au domaine

Procédure pour joindre un poste Windows 10/11 au domaine Active Directory :

Étape par étape

  1. Configurer le DNS : sur le poste client, mettre l'adresse IP du DC (192.168.1.10) comme serveur DNS préféré
  2. Ouvrir les paramètres : Paramètres > Système > À propos > Renommer ce PC (avancé)
  3. Modifier le domaine : cliquer sur "Modifier", cocher "Domaine", entrer entreprise.local
  4. Authentification : saisir les identifiants d'un compte administrateur du domaine
  5. Redémarrer : le poste redémarrera pour finaliser la jonction
  6. Connexion : à l'écran de login, sélectionner "Autre utilisateur" et se connecter avec un compte AD

Si la jonction au domaine échoue, vérifiez en priorité : le DNS du poste client (doit pointer vers le DC), la connectivité réseau, et le pare-feu Windows.

8. Déploiement d'image (Sysprep)

Sysprep (System Preparation Tool) permet de généraliser une installation Windows afin de la dupliquer sur plusieurs machines sans conflit d'identifiants (SID).

Procédure de création d'image

  1. Installer un Windows modèle : installation propre de Windows 10/11
  2. Installer les logiciels : déployer les applications nécessaires (Office, navigateurs, outils métier)
  3. Configurer le système : paramètres, profil par défaut, optimisations
  4. Lancer Sysprep :
Commande Sysprep — généralisation de l'imagesysprep /generalize /oobe /shutdown
ParamètreRôle
/generalizeSupprime les informations uniques (SID, nom machine, pilotes spécifiques)
/oobeRedémarre en mode "Out of Box Experience" (premier démarrage)
/shutdownÉteint le poste après généralisation (prêt pour capture)
  1. Capturer l'image : utiliser un outil de capture
OutilDescription
WDSWindows Deployment Services (role Windows Server)
MDTMicrosoft Deployment Toolkit (déploiement avancé)
ClonezillaOutil open-source de clonage disque
PackerAutomatisation de création d'images (HashiCorp)
  1. Déployer l'image sur les autres postes du parc

N'exécutez jamais Sysprep plus de 8 fois sur une même installation Windows. Au-delà, le compteur interne atteint sa limite et l'opération échoue.

10. Procédures d'administration

Créer un utilisateur

  1. Ouvrir dsa.msc (Active Directory Users and Computers)
  2. Naviguer vers l'OU cible (ex: CPMS > Utilisateurs > IT)
  3. Clic droit > New > User
  4. Renseigner : prénom, nom, identifiant de connexion
  5. Définir le mot de passe et les options (changement au premier login, etc.)
  6. Valider la création

Réinitialiser un mot de passe

  1. Ouvrir dsa.msc
  2. Rechercher l'utilisateur (clic droit sur le domaine > Find)
  3. Clic droit sur l'utilisateur > Reset Password
  4. Saisir le nouveau mot de passe et cocher "User must change password at next logon"

Déplacer un ordinateur dans une OU

  1. Dans dsa.msc, localiser l'ordinateur dans Computers (conteneur par défaut)
  2. Clic droit > Move
  3. Sélectionner l'OU de destination (ex: CPMS > Ordinateurs > Laptops)
  4. Valider

Ajouter un utilisateur à un groupe

  1. Dans dsa.msc, ouvrir les propriétés de l'utilisateur
  2. Onglet "Member Of"
  3. Cliquer Add et rechercher le groupe
  4. Valider

Groupes courants

GroupeDescription
Domain UsersTous les utilisateurs du domaine (groupe par défaut)
Domain AdminsAdministrateurs du domaine (accès total)
IT SupportÉquipe support informatique (groupe personnalisé)

11. Sécurité

Bonnes pratiques de sécurisation AD

MesureDétail
Minimum 2 DCToujours avoir au moins deux contrôleurs de domaine pour la redondance
Sauvegardes régulièresSauvegarder l'AD avec un outil dédié (ex: Veeam Backup)
Comptes admin séparésUtiliser un compte admin dédié (ex: adm-smir) distinct du compte utilisateur quotidien
DNS interne obligatoireNe jamais utiliser de DNS externe en serveur principal sur le DC
Durcissement GPOAppliquer des GPO de sécurité (mots de passe, verrouillage, restrictions)
Désactiver le compte AdministratorDésactiver le compte "Administrator" par défaut et créer des comptes nommés
Politique de mots de passe stricteLongueur minimale 12 caractères, complexité, expiration 90 jours
Supervision SIEMMettre en place un SIEM pour surveiller les événements critiques (connexions échouées, modifications de groupes, etc.)

Ne jamais utiliser le compte Administrator par défaut pour les tâches quotidiennes. Créez des comptes administrateurs nommés et traçables.
