1. Contexte

Le Flipper Zero est un outil de pentest hardware multi-protocole conçu pour l'analyse et le test de sécurité des systèmes physiques et sans fil. Ce projet s'inscrit dans une approche pédagogique de la sécurité matérielle, mené dans le cadre du BTS SIO et de la veille technologique en cybersécurité.

L'objectif est d'explorer les vulnérabilités des protocoles de communication couramment utilisés (RFID, NFC, Sub-GHz, IR) afin de mieux comprendre les risques et de proposer des mesures de remédiation adaptées.

2. Présentation du Flipper Zero

Le Flipper Zero est un appareil portable de pentest multi-protocole, compact et autonome, conçu pour les professionnels et passionnés de sécurité informatique.

Caractéristiques matérielles

  • Écran LCD 1.4" monochrome (128x64 pixels) avec rétro-éclairage
  • Batterie rechargeable Li-Po 2000 mAh (autonomie ~30 jours en veille)
  • Port USB-C pour charge, communication et mode BadUSB
  • Slot microSD pour stockage étendu (firmwares, captures, bases de données)
  • Interface de navigation a 5 boutons + bouton retour

Protocoles supportés

  • Sub-GHz (433/868 MHz) — Capture et replay de signaux radio
  • RFID 125 kHz — Lecture et émulation de badges d'accès
  • NFC 13.56 MHz — Lecture de cartes, émulation UID
  • Infrarouge — Télécommandes universelles, capture/replay
  • GPIO — Interface hardware (UART, SPI, I2C)
  • Bluetooth — Scan BLE, analyse de périphériques
  • USB (BadUSB) — Injection de frappes clavier via scripts

4. Firmwares alternatifs

Firmware officiel vs alternatives

Le firmware officiel du Flipper Zero offre une base stable et régulièrement mise à jour. Cependant, la communauté a développé plusieurs firmwares alternatifs qui étendent considérablement les fonctionnalités de l'appareil.

Firmware Spécificités Avantages
Officiel Stable, support garanti, mises à jour régulières Fiabilité, documentation officielle
Unleashed Suppression des restrictions régionales Sub-GHz Accès à toutes les fréquences, applications supplémentaires
Xtreme Interface personnalisée, animations, packs d'applications Expérience utilisateur enrichie, nombreuses apps pré-installées
RogueMaster Fusion de multiples sources, protocoles étendus Maximum de fonctionnalités, communauté active

Fonctionnalités étendues

  • Protocoles supplémentaires et fréquences déverrouillées
  • Applications custom développées par la communauté
  • Animations et thèmes personnalisés
  • Outils de debug et d'analyse avancés

Processus de flash et mise a jour

  • Flash via l'application desktop qFlipper ou via le navigateur web
  • Sauvegarde préalable des données sur microSD
  • Possibilité de revenir au firmware officiel à tout moment
  • Mise a jour OTA (Over-The-Air) via Bluetooth pour le firmware officiel

5. Cas d'usage pentest interne

Test de badges d'accès RFID/NFC

  • Lecture et identification du type de badge (EM4100, Mifare Classic, HID)
  • Clonage de badges RFID 125 kHz sur des cartes vierges
  • Émulation de l'UID NFC pour tester les contrôles d'accès
  • Analyse des données stockées sur les cartes NFC (secteurs Mifare)
  • Identification des systèmes vulnérables au clonage

Analyse de signaux radio

  • Capture de signaux Sub-GHz de portails et télécommandes
  • Analyse de la modulation et du protocole utilisé (AM/FM, OOK, FSK)
  • Test de replay sur systèmes sans rolling code
  • Détection d'alarmes et capteurs IoT sur les bandes 433/868 MHz
  • Cartographie des équipements radio dans un périmètre donné

Test de télécommandes IR

  • Capture des signaux infrarouges de télécommandes existantes
  • Utilisation de la base de données universelle pour tester des appareils
  • Vérification de la sécurité des systèmes domotiques contrôlés par IR

Démonstrations BadUSB

  • Écriture de scripts d'injection HID en syntaxe Rubber Ducky
  • Démonstration de l'ouverture d'un reverse shell en quelques secondes
  • Sensibilisation aux risques des périphériques USB non vérifiés
  • Test des politiques de sécurité USB des postes de travail

Documentation des vulnérabilités

  • Rapport détaillé de chaque vulnérabilité identifiée
  • Classification par sévérité (critique, haute, moyenne, basse)
  • Recommandations de remédiation pour chaque point
  • Preuve de concept (PoC) documentée avec captures et logs

6. Scripts et développement

Scripts personnalisés en FlipperScript

  • Développement d'applications custom au format .fap (Flipper Application Package)
  • Langage C avec le SDK Flipper pour les applications natives
  • Scripts JavaScript pour le moteur d'automatisation intégré
  • Accès aux APIs hardware (GPIO, radio, NFC, IR) via le SDK

Automatisation de tests

  • Scripts de scan automatique multi-protocole
  • Séquences de tests pré-configurées pour les audits
  • Export automatique des résultats sur microSD
  • Chaînes de tests reproductibles pour les rapports

Intégration avec d'autres outils

  • Wireshark — Analyse des captures radio et NFC exportées
  • SDR++ — Analyse spectrale complémentaire des signaux Sub-GHz
  • Proxmark3 — Analyse RFID/NFC avancée en complément
  • GNU Radio — Traitement de signal avancé
  • Scripts Python pour le post-traitement des données capturées

7. Éthique et cadre légal

Tests uniquement autorisés

Tous les tests sont réalisés exclusivement sur du matériel personnel ou avec une autorisation écrite explicite du propriétaire. Les tests sont menés dans un environnement contrôlé (lab, équipements dédiés) afin d'éviter toute interférence avec des systèmes tiers.

Cadre pédagogique BTS SIO

Ce projet s'inscrit dans le programme du BTS SIO option SLAM/SISR, notamment dans les compétences liées à la cybersécurité et à la veille technologique. L'objectif est de comprendre les vulnérabilités pour mieux s'en protéger.

Documentation pour remédiation

  • Chaque vulnérabilité identifiée est documentée avec une recommandation de correction
  • Les rapports suivent un format standardisé (description, impact, remédiation)
  • L'objectif n'est pas l'exploitation mais la sensibilisation et l'amélioration de la sécurité

Respect du cadre légal français

  • Art. 323-1 Code pénal — Accès frauduleux à un système (interdit sans autorisation)
  • Art. 323-2 Code pénal — Entrave au fonctionnement d'un système
  • Art. 323-3 Code pénal — Introduction, modification ou suppression de données
  • Art. 323-3-1 Code pénal — Détention d'outils de piratage (usage légitime requis)
  • Art. 323-7 Code pénal — Tentative des délits prévus
  • Le Flipper Zero est un outil légal ; son utilisation malveillante ne l'est pas
  • Tous les tests sont documentés et réalisés dans un cadre pédagogique ou professionnel autorisé
