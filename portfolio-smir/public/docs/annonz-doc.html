Annonz — Plateforme d'annonces

Date : 2025
Auteur : S. MIR
Type : Projet scolaire BTS SIO
PHP MySQL Full-text Search Auth CRUD MVC

1. Contexte

Annonz est une plateforme d'annonces full-stack développée dans le cadre du BTS SIO. Il s'agit d'une application web complète permettant aux utilisateurs de publier, rechercher et gérer des annonces en ligne.

Le projet met en œuvre une architecture MVC manuelle en PHP natif, sans framework, afin de maîtriser les fondamentaux du développement web backend et de comprendre les mécanismes sous-jacents habituellement abstraits par les frameworks.

2. Architecture technique

Backend

  • PHP 8.x natif (sans framework) pour une maîtrise complète du code
  • Architecture MVC manuelle avec séparation stricte des responsabilités
  • Routing personnalisé via .htaccess et un front controller
  • PDO pour l'accès aux données avec prepared statements

Base de donnees

  • MySQL avec moteur InnoDB pour les transactions et l'intégrité référentielle
  • Index FULLTEXT pour la recherche avancée par mots-clés
  • Relations et contraintes de clés étrangères

Frontend

  • HTML5 sémantique pour la structure des pages
  • CSS3 avec design responsive (media queries, flexbox, grid)
  • JavaScript vanilla pour les interactions dynamiques (validation, AJAX)

Serveur

  • Compatible Apache (avec mod_rewrite) et Nginx
  • Configuration de réécriture d'URL pour des routes propres

3. Fonctionnalites

a. Gestion des annonces

  • CRUD complet — Création, lecture, modification et suppression d'annonces
  • Attribution de catégories pour organiser les annonces
  • Upload d'images pour illustrer chaque annonce
  • Statut de l'annonce (active, en attente, archivée)
  • Pagination des résultats pour une navigation fluide

b. Recherche full-text

  • Index MySQL FULLTEXT sur les colonnes titre et description
  • Recherche par mots-clés avec classement par pertinence (MATCH ... AGAINST)
  • Filtrage par catégorie, fourchette de prix et localisation
  • Tri des résultats par pertinence, date de publication ou prix
  • Suggestions de recherche basées sur les termes populaires

c. Authentification

  • Inscription avec validation des champs (email, mot de passe fort)
  • Connexion sécurisée avec gestion de sessions PHP
  • Hashage des mots de passe avec bcrypt via password_hash()
  • Validation de l'adresse email lors de l'inscription
  • Gestion des rôles (utilisateur, administrateur)
  • Déconnexion avec destruction complète de la session

d. Upload sécurisé

  • Validation du type MIME réel du fichier (pas seulement l'extension)
  • Limite de taille configurable (ex. 2 Mo maximum par image)
  • Renommage sécurisé avec uniqid() pour éviter les collisions et injections
  • Stockage organisé dans des répertoires structurés par date
  • Whitelist d'extensions autorisées (jpg, jpeg, png, webp)

e. Moderation

  • Système de signalement d'annonces par les utilisateurs
  • Interface d'administration pour la validation des annonces signalées
  • Gestion des utilisateurs (suspension, suppression de compte)
  • Tableau de bord administrateur avec statistiques

f. Messagerie

  • Système de contact entre utilisateurs via l'annonce
  • Formulaire de contact lié à chaque annonce
  • Boîte de réception pour consulter les messages reçus
  • Notification de nouveaux messages

5. Sécurité

Protection contre l'injection SQL

Toutes les requêtes sont exécutées via des prepared statements PDO. Les paramètres utilisateurs ne sont jamais concaténés directement dans les requêtes SQL, éliminant ainsi le risque d'injection.

Protection XSS

  • Échappement systématique des sorties avec htmlspecialchars()
  • En-têtes Content-Security-Policy (CSP) pour limiter les sources de scripts
  • Validation et sanitization des entrées utilisateur

Protection CSRF

  • Génération de tokens CSRF uniques par session
  • Vérification du token sur chaque formulaire soumis en POST
  • Renouvellement du token après chaque action sensible

Hashage des mots de passe

  • Utilisation de password_hash() avec l'algorithme bcrypt
  • Vérification via password_verify() sans jamais comparer en clair
  • Coût de hashage configuré pour un bon équilibre sécurité/performance

Validation côté serveur

  • Toutes les entrées utilisateur sont validées côté serveur (jamais uniquement côté client)
  • Vérification des types, longueurs, formats et plages de valeurs
  • Messages d'erreur génériques pour ne pas divulguer d'informations sensibles

Upload sécurisé

  • Whitelist stricte d'extensions autorisées (.jpg, .jpeg, .png, .webp)
  • Vérification du type MIME réel avec finfo_file()
  • Stockage en dehors du répertoire web accessible quand possible

6. Architecture MVC

L'architecture Modèle-Vue-Contrôleur (MVC) permet de séparer clairement les responsabilités au sein de l'application, facilitant la maintenance et l'évolution du code.

Models — Accès aux données

  • Classes PHP dédiées à l'interaction avec la base de données via PDO
  • Un modèle par entité (UserModel, AnnonceModel, CategoryModel, MessageModel)
  • Méthodes CRUD standardisées (find, findAll, create, update, delete)
  • Encapsulation de la logique d'accès aux données

Views — Templates HTML/PHP

  • Fichiers PHP mélangeant HTML et données transmises par le contrôleur
  • Système de layout avec header/footer partagés
  • Échappement systématique des variables affichées
  • Templates réutilisables pour les composants récurrents (cards, formulaires)

Controllers — Logique métier et routing

  • Front controller (index.php) recevant toutes les requêtes
  • Routeur analysant l'URL pour dispatcher vers le bon contrôleur/méthode
  • Chaque contrôleur gère un domaine fonctionnel (AuthController, AnnonceController, AdminController)
  • Validation des entrées, appel aux modèles, transmission des données aux vues
  • Middlewares pour la verification d'authentification et d'autorisation

Structure des répertoires

Répertoire Contenu

/app/views/Templates HTML/PHP
