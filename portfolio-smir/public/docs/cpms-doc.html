1. Contexte et problematique

La CPMS (Caisse de Prevoyance Mutuelle Sante) gere plusieurs systemes d'information heterogenes pour l'administration de ses 180+ collaborateurs :

  • Active Directory (WAD) — Authentification Windows, messagerie, acces reseau
  • AS400 (IBM i) — Systeme historique de gestion RH, paie et metier
  • Applications ODBC — Outils metier connectes en ODBC au AS400

Avant le projet

  • Changement de mot de passe : procedure manuelle necessitant l'intervention du service IT
  • Delai moyen de traitement : 24 a 48 heures
  • 3 systemes distincts a mettre a jour separement (AD, AS400, applications ODBC)
  • Surcharge du helpdesk : les demandes de reinitialisation representaient une part majeure des tickets
  • Aucune autonomie utilisateur en dehors des heures de bureau

Apres le projet

  • Self-service complet : changement de mot de passe et deverrouillage de compte sans intervention IT
  • Synchronisation temps reel sur tous les systemes : < 30 secondes
  • Disponibilite 24h/24, 7j/7
  • Interface unifiee : un seul point d'entree pour toutes les operations

3. Fonctionnalites detaillees

3.1 Changement de mot de passe multi-systemes

Fonctionnalite centrale du projet : un changement de mot de passe unifie qui synchronise simultanement l'Active Directory (WAD) et le systeme AS400, le tout en une seule action utilisateur.

Identification de l'utilisateur

  • Accepte trois formats d'identifiant : SAMAccountName, adresse email et UPN (User Principal Name)
  • Recherche automatique dans l'annuaire Active Directory via LdapRecord
  • Verification de l'existence et de l'etat du compte avant traitement

Politique de mot de passe

CritereExigence
Longueur minimale15 caracteres
MajusculeAu moins 1 lettre majuscule
MinusculeAu moins 1 lettre minuscule
ChiffreAu moins 1 chiffre
Caractere specialAu moins 1 caractere special (!@#$%...)

Execution technique

  • Etape 1 : Validation du formulaire cote client (Vue.js) + cote serveur (Laravel)
  • Etape 2 : Changement du mot de passe Active Directory via LDAPS (port 636)
  • Etape 3 : Appel du script Python change_password.py via Process::run() pour le changement AS400
  • Etape 4 : Enregistrement en base de donnees (audit log)
  • Temps d'execution total : 5 a 15 secondes

3.2 Reactivation de compte par questions de securite

Permet a un utilisateur dont le compte est verrouille de le deverrouiller sans contacter le helpdesk, via un systeme de challenge par questions de securite.

Enregistrement de la question de securite

  • L'utilisateur definit sa propre question et sa reponse personnalisee
  • La reponse est hashee en bcrypt avec sel individuel avant stockage
  • Possibilite de modifier la question a tout moment depuis le profil

Processus de challenge

  • Presentation de 10 questions : 1 question reelle + 9 questions leurres generees aleatoirement
  • L'utilisateur doit identifier sa question parmi les leurres, puis fournir la bonne reponse
  • Double verification : selection de la bonne question ET reponse correcte
  • Ce mecanisme empeche les tentatives de brute force (1 chance sur 10 de deviner la bonne question)

Limites et securite

ParametreValeurObjectif
Tentatives reussies / jour3 maxLimiter l'abus du self-service
Tentatives echouees / jour3 maxProtection anti brute force
Tracking IPOui, sur chaque tentativeAudit et detection d'anomalies
JournalisationAudit completTracabilite de chaque action
  • Temps de deverrouillage : environ 30 secondes
  • En cas de depassement des limites, l'utilisateur est redirige vers le helpdesk

3.3 Tableau de bord administrateur

Interface dediee aux administrateurs IT pour le monitoring temps reel et la gestion des comptes utilisateurs.

Statistiques en temps reel

  • Nombre de comptes verrouilles actuellement dans l'Active Directory
  • Nombre de deverrouillages effectues dans les dernieres 24h
  • Nombre de tentatives echouees (avec details)
  • Graphique 7 jours : evolution des verrouillages et deverrouillages

Gestion des comptes

  • Recherche avancee avec filtres : comptes verrouilles, comptes desactives, par OU (Unite Organisationnelle), par departement
  • Deverrouillage manuel d'un compte depuis l'interface
  • Historique complet de chaque compte : tentatives, verrouillages, deverrouillages
  • Top des echecs par utilisateur et par adresse IP

Audit et journalisation

  • Logs d'audit complets avec pagination
  • Filtrage par date, type d'action, utilisateur, adresse IP
  • Export des donnees pour analyse

Authentification admin

  • Authentification par Bearer token (genere via endpoint dedie)
  • Duree de vie du token : 8 heures
  • Rotation automatique a l'expiration
  • Permissions basees sur l'appartenance au groupe AD administrateurs

6. Securite

Chiffrement des communications

  • Connexion Active Directory exclusivement en LDAPS (port 636, TLS)
  • API servie en HTTPS avec certificat SSL
  • Aucune donnee sensible transmise en clair

Protection des donnees

  • Reponses aux questions de securite hashees en bcrypt avec sel individuel
  • Aucun mot de passe stocke en base de donnees
  • Tokens admin avec duree de vie limitee (8 heures)
  • Sanitisation de toutes les entrees utilisateur (XSS, injection SQL)

Protection anti brute force

  • Rate limiting : maximum 3 tentatives reussies + 3 tentatives echouees par jour et par utilisateur
  • Tracking IP : adresse IP enregistree sur chaque tentative
  • Questions leurres : 9 fausses questions melangees avec la vraie pour empecher la deduction
  • Blocage automatique au-dela des limites quotidiennes

Audit et tracabilite

  • Journal d'audit complet : chaque action est tracee (qui, quoi, quand, depuis quelle IP)
  • Logs horodates et immuables
  • Consultation par les administrateurs via le tableau de bord
  • Possibilite d'export pour analyse forensique
