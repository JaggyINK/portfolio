1. Contexte et objectifs

Earth Sanitation est une entreprise spécialisée dans les services de plomberie d'urgence, de débouchage et d'assainissement. Elle intervient 24 heures sur 24, 7 jours sur 7, y compris les jours fériés, sur un rayon de 100 km autour de Montpellier et Nîmes, couvrant 3 départements (Hérault 34, Gard 30, Aude 11) et 59 villes.

Objectifs du projet

  • Vitrine professionnelle : créer un site de présentation moderne et rassurant pour les particuliers et les professionnels recherchant des interventions d'urgence en assainissement.
  • Génération de leads : mettre en place un formulaire de contact et devis avec upload de photos, connecte a une base de données, permettant le suivi des demandes depuis un back-office admin.
  • SEO local agressif : dominer les résultats de recherche locaux grâce à 59 pages géolocalisées dynamiques (/zone/[ville]), 7 pages de services dédiées, des données structurées Schema.org (7 types), et un sitemap dynamique généré depuis Prisma.
  • Dashboard d'administration : permettre au client de gérer ses leads, avis clients, articles de blog, FAQ, et de suivre ses statistiques de visites et d'appels depuis une interface protégée.
  • Suivi de présence en ligne : outil intégré de vérification de présence sur 24 plateformes (Google Business, Pages Jaunes, etc.).
  • Conversion maximale : CTA omnipresents (téléphone, formulaire, devis), boutons d'appel direct, signaux de confiance (avis, certifications, expérience).

Cadre BTS SIO : Ce projet freelance mobilise les compétences des blocs SLAM (développement fullstack, conception BDD, API REST, authentification, déploiement) et SISR (sécurité des accès, headers HTTP, hébergement). Il illustre la capacité à livrer un produit complet à un client réel dans un contexte professionnel.

4. Dashboard Admin

Interface d'administration complète, protégée par authentification JWT (NextAuth) avec middleware withAuth sur toutes les routes /admin/*.

4.1 Dashboard principal — Statistiques

Page d'accueil de l'admin avec vue d'ensemble de l'activité du site :

  • 4 StatCards : visites totales, appels téléphoniques, leads reçus, taux de conversion
  • Tendances hebdomadaires : pourcentage d'évolution par rapport à la semaine précédente (flèche verte/rouge)
  • Graphiques : courbes de visites et de leads sur les 30 derniers jours
  • API : GET /api/stats — agrège les données depuis les tables PageVisit, PhoneClick, Lead

4.2 Gestion des Leads

  • Liste structurée : cartes avec informations complètes (nom, téléphone, email, service, message, type, date)
  • Changement de statut : nouveau → en cours → traité → archivé
  • Lightbox photos : visualisation grand format des photos uploadées par le client
  • Export CSV : téléchargement de tous les leads au format CSV
  • Suppression : suppression individuelle avec confirmation
  • API : GET/PUT/DELETE /api/leads + /api/leads/[id]

4.3 Vérificateur de présence en ligne

  • 24 plateformes vérifiées : Google Business Profile, Pages Jaunes, Yelp, TripAdvisor, Facebook, etc.
  • Vérification automatique ou manuelle de la présence de l'entreprise sur chaque plateforme
  • Indicateurs visuels (présent / absent / à vérifier)
  • Liens directs vers les pages de création de profil

4.4 Avis clients (CRUD)

  • Création : ajout d'avis avec nom, note, commentaire, source (Google, Pages Jaunes, site, etc.)
  • Édition : modification de tous les champs
  • Visibilité : toggle visible/masqué (les avis masqués n'apparaissent pas sur le site public)
  • Réordonnancement : choix de l'ordre d'affichage des avis sur le site
  • Sources multiples : indication de la provenance de l'avis
  • Suppression : avec confirmation
  • API : GET/POST /api/reviews + GET/PUT/DELETE /api/reviews/[id]

4.5 Blog admin (CRUD)

  • Création d'article : titre, contenu (rich text), slug, méta description, image
  • Édition : modification complète de l'article
  • Activation / Désactivation : toggle pour publier ou dépublier un article sans le supprimer
  • Suppression : avec confirmation
  • API : GET/POST /api/blog + GET/PUT/DELETE /api/blog/[id]

4.6 FAQ admin (CRUD)

  • Création : question, réponse, service associé (ou général)
  • Organisation par service : chaque FAQ est liée à un service spécifique ou à la page FAQ générale
  • Réordonnancement : modification de l'ordre d'affichage
  • Suppression : avec confirmation
  • API : GET/POST /api/faq + GET/PUT/DELETE /api/faq/[id]

4.7 Paramètres

  • Changement de mot de passe : ancien mot de passe + nouveau mot de passe + confirmation
  • Validation côté serveur, hachage bcrypt
  • API : POST /api/admin/change-password

4.8 Sécurité du dashboard

  • Middleware withAuth : vérifie la session JWT sur toutes les routes /admin/*
  • JWT (NextAuth) : tokens signés, expiration configurable
  • CSRF : protection intégrée via NextAuth
  • Rate limiting login : 5 tentatives max par 15 minutes
  • Mots de passe : hachés avec bcrypt (coût 10+)

6. SEO et indexation

Stratégie SEO mise en place

Sitemap dynamique

  • Généré automatiquement depuis Prisma (pages statiques + services + 59 zones + articles blog)
  • Mis à jour à chaque build avec les dernières données BDD
  • Priorités différenciées (homepage: 1.0, services: 0.9, zones: 0.8, blog: 0.7)

robots.txt

  • Autorise tous les crawlers sur les pages publiques
  • Bloque l'accès à /admin, /api
  • Référence le sitemap

Web App Manifest (PWA-ready)

  • Nom, description, icônes, couleur de thème
  • Permet l'installation sur mobile comme une application native

Données structurées Schema.org(7 types)

Type de schémaUtilisé surDonnées clés

BreadcrumbSchema59 pages zones + servicesFil d'Ariane structuré (Accueil → Zone → Ville)

Balises méta et Open Graph

  • Titre unique par page (optimisé avec mots-clés locaux)
  • Meta description unique par page
  • Canonical URL sur chaque page pour éviter le contenu dupliqué
  • Open Graph : og:title, og:description, og:image, og:url, og:type sur toutes les pages
  • Twitter Cards : twitter:card, twitter:title, twitter:description, twitter:image
  • Keywords : mots-clés ciblés par page (service + ville + département)
