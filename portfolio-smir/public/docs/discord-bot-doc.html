1. Contexte et objectifs

Bot Discord concu pour automatiser la veille technologique dans le cadre du BTS SIO. Il assure la publication automatique, 24 h/24, d'articles issus de sources reconnues en securite informatique et developpement web.

L'objectif est de centraliser la collecte d'informations techniques dans un serveur Discord dedie, permettant aux etudiants et professionnels de rester informes en temps reel sans effort manuel.

2. Architecture technique

Stack technique

  • Python 3 — Langage principal
  • discord.py — Bibliotheque d'interaction avec l'API Discord
  • asyncio — Programmation asynchrone pour les taches concurrentes
  • feedparser — Parsing des flux RSS/Atom
  • aiohttp — Requetes HTTP asynchrones

Paradigme

Architecture event-driven reposant sur des commandes slash et des listeners d'evenements Discord. Les taches de collecte RSS s'executent en arriere-plan via des boucles asynchrones, garantissant une reactivite constante du bot.

5. Fonctionnalites

  • Agregation multi-sources RSS — Collecte unifiee depuis l'ensemble des flux configures
  • Filtrage par mots-cles — Tri intelligent des articles selon les thematiques pertinentes
  • Publication automatique — Envoi dans les channels dedies sans intervention humaine
  • Formatage rich embed Discord — Mise en forme avancee avec titres, descriptions, miniatures et liens
  • Gestion des erreurs et retry — Mecanismes de reprise automatique en cas d'echec de requete
  • Logs d'activite — Journalisation complete des actions du bot pour le suivi et le debug

6. Securite

  • Token bot securise — Stockage du token via variables d'environnement (.env), jamais en dur dans le code
  • Rate limiting Discord API — Respect des limites imposees par l'API Discord pour eviter les bans
  • Validation des sources RSS — Verification de l'integrite et de la disponibilite des flux avant traitement
  • Sanitization des contenus — Nettoyage des donnees recues pour prevenir les injections dans les messages
