1. Contexte et objectifs

Bot Discord conçu pour automatiser la veille technologique dans le cadre du BTS SIO. Il assure la publication automatique, 24 h/24, d'articles issus de sources reconnues en sécurité informatique et développement web.

L'objectif est de centraliser la collecte d'informations techniques dans un serveur Discord dédié, permettant aux étudiants et professionnels de rester informés en temps réel sans effort manuel.

2. Architecture technique

Stack technique

  • Python 3 — Langage principal
  • discord.py — Bibliothèque d'interaction avec l'API Discord
  • asyncio — Programmation asynchrone pour les tâches concurrentes
  • feedparser — Parsing des flux RSS/Atom
  • aiohttp — Requêtes HTTP asynchrones

Paradigme

Architecture event-driven reposant sur des commandes slash et des listeners d'événements Discord. Les tâches de collecte RSS s'exécutent en arrière-plan via des boucles asynchrones, garantissant une réactivité constante du bot.

5. Fonctionnalités

  • Agrégation multi-sources RSS — Collecte unifiée depuis l'ensemble des flux configurés
  • Filtrage par mots-clés — Tri intelligent des articles selon les thématiques pertinentes
  • Publication automatique — Envoi dans les channels dédiés sans intervention humaine
  • Formatage rich embed Discord — Mise en forme avancée avec titres, descriptions, miniatures et liens
  • Gestion des erreurs et retry — Mécanismes de reprise automatique en cas d'échec de requête
  • Logs d'activité — Journalisation complète des actions du bot pour le suivi et le debug

6. Sécurité

  • Token bot sécurisé — Stockage du token via variables d'environnement (.env), jamais en dur dans le code
  • Rate limiting Discord API — Respect des limites imposées par l'API Discord pour éviter les bans
  • Validation des sources RSS — Vérification de l'intégrité et de la disponibilité des flux avant traitement
  • Sanitization des contenus — Nettoyage des données reçues pour prévenir les injections dans les messages
