2. Contexte et objectifs

ContexteCréer un portfolio développeur qui se démarque en combinant une présentation classique avec une expérience 3D interactive explorant une surface lunaire avec des stations de quiz.
Objectif principalDévelopper un portfolio dual : mode classique (sections scrollables) + mode 3D (exploration lunaire interactive avec quiz techniques).
Objectifs secondaires
  • Scène 3D temps réel avec 40+ composants Three.js
  • Physique lunaire réaliste (gravité, inertie, sauts)
  • 5 stations de quiz interactives (PHP, Python, JS, SQL, Docker)
  • Portfolio classique avec 9 sections complètes
  • Responsive design et optimisation WebGL

3. Périmètre fonctionnel

FonctionnalitéDescription
Mode classiquePortfolio scrollable : hero, à propos, BTS SIO, parcours, projets, certifications, veille, écoles.
Mode 3DScène lunaire interactive : terrain procédural, astronaute, bases lunaires, astéroïdes, nébuleuses.
Quiz stations5 stations sur la lune : quiz PHP, Python, JavaScript, SQL, Docker avec scoring.
NavigationWASD/flèches + souris drag (inertie). Saut avec physique lunaire.
ResponsiveAdapté mobile/desktop. DPR dynamique, lazy loading des éléments 3D lointains.
AccessibilitéSupport prefers-reduced-motion, navigation clavier complète.

4. Architecture technique

ComposantTechnologie
FrameworkReact 19 + React Router 6 — Architecture dual-layer
3D EngineThree.js 0.180 + React Three Fiber 9 + Drei 10
BuildVite 5 avec manual chunking (MoonScene séparé)
StylesTailwind CSS 3.4 + thème centralisé (PHI/golden ratio)
PhysiqueRefs-based physics (quaternions, inertie, gravité) dans useFrame
StateContext API + localStorage. Custom events pour React ↔ Three.js

5. Contraintes

ContrainteDétail
Performance60 FPS constant. DPR max 1.75x, lazy loading des éléments décoratifs.
CompatibilitéFonctionne sur mobile et desktop. Fallback si WebGL non supporté.
Taille bundleChunking manuel pour éviter un bundle monolithique (Three.js ~ 600KB).
