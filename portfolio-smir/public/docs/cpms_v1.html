1. Contexte et objectifs

L'Intranet CPMS V1 est un portail numérique interne conçu pour la Caisse de Prévoyance Mutuelle Santé (CPMS). Cette première version représente une application modulaire avec une interface moderne, permettant aux collaborateurs de gérer leurs mots de passe, débloquer leurs comptes et consulter une FAQ interactive.

Le projet s'inscrit dans le cadre du BTS SIO option SLAM, épreuve E5 (Conception et maintenance de solutions informatiques). Il illustre la maîtrise complète du cyclé de développement d'une application web full-stack moderne, de la conception à lamise en production, incluant l'intégration de multiples systèmes d'entreprise (Active Directory, AS400, LDAPS).

Objectifs fonctionnels

  • Centralisation documentaire : regrouper tous les documents RH avec visionneuse PDF intégrée, recherche, filtres par tags et deep linking
  • Self-service mot de passe : permettre le changement de mot de passe sur trois systèmes simultanément (Active Directory, AS400, ODBC)
  • Déblocage de compte : offrir un mécanisme autonome de déblocage de compte AD via questions de sécurité
  • Administration centralisée : fournir un tableau de bord aux administrateurs IT pour gérer les comptes AD verrouillés/suspendus
  • FAQ interactive : proposer un chatbot et un accordeon FAQ couvrant 50 questions en 11 catégories

Public cible

  • Collaborateurs CPMS : tous les salariés de l'entreprise, tout profil technique confondu
  • Administrateurs IT : équipe technique utilisant le dashboard admin pour la gestion des comptes AD
  • Managers : encadrants consultant les tutoriels vidéo et les documents internes

8. Sécurité

Systeme d'authentification admin

SystemeMethodeStockage tokenMiddleware
Admin ITAD + groupe Intranet-Adminsadmin_tokenAdminApiAuth
  • Validation triple : le middleware vérifié le token, l'adresse IP et le User-Agent
  • CSRF Sanctum : les operations publiques utilisent le cookie CSRF (X-XSRF-TOKEN)
  • Auto-logout : redirection automatique sur 401 (token expire)
  • Isolation credentials : credentials: 'omit' pour admin, credentials: 'include' pour les routes CSRF

Protection contre le brute-force

MecanismeConfigurationEndpoint
Rate limiting5 requêtes / minutePOST /change-password
Rate limiting3 requêtes / 10 minutesPOST /unlock-account
Rate limiting3 requêtes / 10 minutesPOST /security-questions/*
Rate limiting10 requêtes / minutePOST /admin/login
Limite quotidienne3 succes + 3 echecs / jourDéblocage self-service
Timer frontendCountdown 60s sur 429Tous les formulaires

Sécurité LDAP et AS400

  • LDAPS : connexion chiffrée TLS sur le port 636
  • Comptes de service dédiés : principe du moindre privilege (un compte pour reset, un pour unlock)
  • Variable d'environnement : mot de passe AS400 passe via AS400_OLD_PASSWORD (jamais en argument)
  • Echappement : identifiant et mot de passe echappes via escapeshellarg()
